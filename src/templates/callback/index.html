The callback query string included:

  • The state we provided on the first page (usually used for tracking sessions ): 
    {{ state }}
  • The code we need to exchange for a access token in the back-channel: 
    {{ code }}

The server made the token request to Salesforce retrieving:

  • Instance url: 
    {{ instance_url }}
  • Access token: 
    {{ access_token }}
  • Refresh token: 
    {{ refresh_token }}
  • SF User ID URL: 
    {{ id }}
  • Token issue timestamp: 
    {{ issued_at }}
Additional documentation about what those mean can be found here, but the TL;DR is that the access token can be used to gain access to the logged in user's Salesforce data and the refresh token can be used to generate additional access tokens.
